You can add styling rules in this box, using the full capabilities of CSS.

For example, if you add the rule .today .date {color: red;} the date for today's events will be rendered in red.

In your CSS selector in place of .today you may use .nottoday, .tomorrow, .future, .nextweek, or an English-language weekday name like .tuesday. .future refers to all days except today and tomorrow. .nextweek refers to the day that's one week from today.

If you want to show only an overview of future events, you can use the following style rule. It conceals locations and descriptions for all events except today's and tomorrow's. 

.future .description, .future .location {display: none;}

For today's events, you can use the .current and .past selectors. For example, to show current (presently in progress) events in orange and completed events in gray, use these style rules. 

.current .all {color: orange;}
.past .all {color: gray;}

You can use .day1 to style the first day of each month, or .day25 to style the 25th day. You can use the .december selector in the same way to style everything in December. You can combine these selectors. For example, .december.day25 styles events on December 25th.

In place of .date you may use .time, .summary, .location or .description. If you use .header you'll style the time, summary, and location all together. If you use .all you'll style all parts of the display. So, you can show the header for tomorrow's events in red like this: 

.tomorrow .header {color:red;}

You can show everything about Sunday events in dark blue like this: 
.sunday .all {color:darkblue;}

To indent the location of each event, and show the location of today's events in blue, you can use this CSS: 

.location {margin-left: 40px; }
.today .location {color:blue; }
